🔑 Connexion & Compte
1ère fois — Créer un compte : prénom, nom, rôle, code à 6 chiffres.
Ensuite — Recherche ton nom → sélectionne → entre ton code. Coche "Se souvenir" pour 3 mois d'auto-connexion.
Nom sélectionné — La recherche se masque automatiquement. Utilise la croix ✕ à droite du nom si tu t'es trompé.
Code oublié ? — Clique "Code oublié ?" → envoie la demande → l'admin te fournit un code provisoire.
Sécurité — Après 3 codes faux, le compte est bloqué. Déblocage uniquement par l'administrateur.
Accès Admin (bouton rouge) — Ouvre le panneau admin en lecture globale (pas de modification des attributions de lits).
👥 Rejoindre la garde
À la connexion : OUI je suis de service → tu rejoins l'effectif de la garde.
NON → mode lecture seule.
Les compteurs IDE / AS se mettent à jour en temps réel. Présence partagée jour ↔ nuit.
🛏️ Assigner les lits
① Tape ton nom dans EFFECTIF → bandeau orange s'affiche.
② Touche les lits à attribuer → ils affichent ton nom.
③ ✔ VALIDER pour confirmer.

📅 Planning & Congés Annuels
Menu ☰ → Planning & Validation CA. Calendrier annuel personnel pour poser et valider ses congés.
Régime ☀ Jour ou ☾ Nuit — le calcul s'adapte (max 21 j consécutifs jour / 22 j nuit, RCN, fractionnement). Coche les DJF travaillés pour le calcul du RCV.
Heures supplémentaires — trois états disponibles : HS (générique) · HSJ (jour) · HSN (nuit) — pour distinguer les HS selon la plage horaire. Tous comptent dans les DJF.
Données sauvegardées localement sur l'appareil (non partagées). Bouton ↺ pour réinitialiser l'année.
🔄 Bourse d'échange
Menu ☰ → Bourse d'échange. Système de permutation de gardes entre collègues, synchronisé en temps réel.
① Proposer — Appuyer sur +, sélectionner la garde à céder (date + Jour/Nuit) et la garde souhaitée (facultatif).
② Consulter — Les demandes ouvertes des collègues s'affichent en temps réel.
③ Accepter — Valide l'échange côté applicatif. La permutation doit ensuite être confirmée avec le cadre de santé.
Connexion obligatoire. Les demandes sont visibles de tous les utilisateurs connectés.

Menu ☰ → Les Services. Annuaire complet des services et numéros de poste internes.
Recherche par nom ou numéro. Les résultats apparaissent également dans la recherche globale en haut du dashboard.

Termes, fonctions et architecture de PulseUnit

📦 Modules de l'application ›
Tableau de bordGrille principale — lits RÉA + USIP, effectifs, assignations par garde
Simulateur Respirateur4 modes ventilatoires (PC, VC, VS-AI, VNI) — waveforms animées — thème clair/sombre — 8 scénarios — modal de modification
Normes & Alertes7 paramètres vitaux interactifs — barre de zones, tableau de référence, analyse clinique, 8 scénarios de simulation
Lexique médical260+ termes en 13 catégories — recherche libre + filtre par catégorie
Calculateurs15 calculateurs — IBW, PAM, P/F, ΔP, PSE, glycémie, diurèse… Aucune sauvegarde entre sessions
ProtocolesProtocoles de soins en lecture seule — accordéon par section
Tâches IDE TechListe auto selon jour/nuit — barre de progression, sauvegardée par garde
Checklist chambre10 items de vérification par lit — sauvegardés par garde
Calendrier CongésSimulateur CA — bonus HS, fractionnement, RCV selon Décrets 2002-8 et 2002-9
⚙️ Termes techniques du code ›
shiftHistoryObjet JS central — historique de toutes les gardes (clé = YYYY-MM-DD-jour/nuit)
rosterTableau des soignants permanents — persist en localStorage
currentShiftKeyClé de garde active — format "2026-04-09-jour"
CONFIGConfiguration des secteurs (RÉA / USIP) et numéros de lits
NORMES_ZONESZones cliniques par paramètre — seuils, labels, couleurs, sévérité
NORMES_SCENARIOS8 scénarios prédéfinis — valeurs simulées pour chaque paramètre vital
escapeHTML()Neutralise les caractères HTML avant injection DOM — protection XSS
isShiftLocked()true si la garde est terminée — JOUR verrouillé à 20h, NUIT à 8h lendemain
saveData()Sauvegarde roster + shiftHistory en localStorage — alerte si quota dépassé
renderApp()Reconstruit tout le tableau de bord — appelée après chaque modification
